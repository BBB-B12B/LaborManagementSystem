# Daily Contractor Data Schema (ข้อมูลแรงงานรายวัน)

เอกสารนี้สรุปรายละเอียดข้อมูลที่จัดเก็บสำหรับ "แรงงานรายวัน" (Daily Contractor) ทั้งหมดในระบบ รวมถึงความจำเป็นและวัตถุประสงค์ของแต่ละสนามข้อมูล

## 1. ข้อมูลทั่วไป (Profile)
**Collection:** `dailyContractors`
ข้อมูลพื้นฐานสำหรับระบุตัวตนและตรวจสอบสิทธิ์

| ชื่อหัวตาราง (Template) | Field (Database) | ความหมาย/วัตถุประสงค์ | ความจำเป็น |
| :--- | :--- | :--- | :--- |
| **รหัสพนักงาน** | `employeeId` | รหัสประจำตัวพนักงาน ใช้สำหรับอ้างอิงทุกระบบ (Scan, Payroll) | **จำเป็น (Key)** |
| **ชื่อ-นามสกุล** | `name` | ชื่อเต็มของพนักงาน | **จำเป็น** |
| **รหัสทักษะ** | `skillId` | รหัสทักษะ (Skill Code) เพื่อระบุประเภทแรงงานและค่าแรงมาตรฐาน | **จำเป็น** |
| **รหัสโครงการ** | `projectLocationIds` | **รหัสโครงการ (Project Code)** เช่น "HO", "WH1" (ไม่ใช่ UUID/System ID) ใส่ได้หลายโครงการ **คั่นด้วยคอมมา (,)** | **จำเป็น** |
| สถานะใช้งาน | `isActive` | สถานะการทำงาน (`true` = ทำงานอยู่, `false` = ลาออก/พักงาน) | จำเป็น (Default=True) |
| วันเริ่มงาน | `startDate` | วันที่เริ่มงาน (ใช้คำนวณอายุงาน/สิทธิ์) | แนะนำ |

*(Field อื่นๆ เช่น เบอร์โทร, เลขบัตรปชช., ที่อยู่, วันสิ้นสุดงาน ถูกตัดออกเพื่อความกระชับของการนำเข้าข้อมูล)*

---

## 2. ข้อมูลรายได้ (Income Details)
**Sub-Collection:** `dailyContractors/{id}/dcIncomeDetails`
ข้อมูลสำหรับคำนวณค่าแรง

| ชื่อหัวตาราง (Template) | Field (Database) | ความหมาย/วัตถุประสงค์ | ความจำเป็น |
| :--- | :--- | :--- | :--- |
| **ค่าแรงต่อชั่วโมง** | `hourlyRate` | ฐานค่าแรงต่อชั่วโมง (ใช้คำนวณ OT โดยอัตโนมัติ x1.5) | **จำเป็น** |

> **💡 วิธีคำนวณ:** หากทราบค่าแรงรายวัน ให้หารด้วย 8 ชั่วโมง
> *   ตัวอย่าง: ค่าแรงวันละ **482** บาท
> *   ค่าแรงต่อชั่วโมง = 482 / 8 = **60.25** บาท
> *   (กรุณากรอก 60.25 ลงในไฟล์)
| ค่าวิชาชีพ | `professionalRate` | ค่าวิชาชีพพิเศษ (On-top) **จ่ายรายวัน** (คูณจำนวนวันที่มาทำงาน) | ทางเลือก |
| ค่าโทรศัพท์ | `phoneAllowance` | ค่าโทรศัพท์เหมาต่องวด | ทางเลือก |
| *คำนวณอัตโนมัติ* | `otHourlyRate` | ระบบคำนวณให้จาก `hourlyRate * 1.5` ไม่ต้องเก็บ | - |

## 3. ข้อมูลรายจ่าย/หัก (Expense Details)
**Sub-Collection:** `dailyContractors/{id}/dcExpenseDetails`
ข้อมูลสำหรับหักค่าใช้จ่ายสวัสดิการต่างๆ

| ชื่อหัวตาราง (Template) | Field (Database) | ความหมาย/วัตถุประสงค์ | ความจำเป็น |
| :--- | :--- | :--- | :--- |
| **ค่าที่พัก** | `accommodationCost` | ค่าที่พักของพนักงานเอง (หักต่องวด) | **จำเป็น (ถ้าพักแคมป์)** |
| จำนวนผู้ติดตาม | `followerCount` | จำนวนผู้ติดตามที่มาพักด้วย (ภรรยา, ลูก) | จำเป็น (เพื่อคำนวณค่าน้ำไฟ) |
| *คำนวณอัตโนมัติ* | `followerAccommodation` | ค่าที่พักผู้ติดตาม (300 บาท x จำนวนคน) | - |
| ค่าตู้เย็น | `refrigeratorCost` | ค่าเช่า/ไฟ ตู้เย็น | ทางเลือก (ตามจริง) |
| ค่าเครื่องเสียง | `soundSystemCost` | ค่าไฟเครื่องเสียง | ทางเลือก (ตามจริง) |
| ค่าทีวี | `tvCost` | ค่าไฟทีวี | ทางเลือก (ตามจริง) |
| ค่าเครื่องซักผ้า | `washingMachineCost` | ค่าไฟเครื่องซักผ้า | ทางเลือก (ตามจริง) |
| ค่าแอร์เคลื่อนที่ | `portableAcCost` | ค่าไฟแอร์เคลื่อนที่ | ทางเลือก (ตามจริง) |

## สรุปคำแนะนำ (Recommendation)
### ✅ สิ่งที่จำเป็นต้องมี (Must Have)
ข้อมูลเหล่านี้หากขาดไป ระบบจะไม่สามารถทำงานหรือคำนวณเงินได้ถูกต้อง:
1.  **Employee ID**: หัวใจสำคัญของระบบ Link ข้อมูลทุกอย่าง
2.  **Name**: เพื่อการระบุตัวตน
3.  **Skill & Project**: เพื่อจัดกลุ่มและลงบัญชีโครงการ (ใช้รหัสย่อเช่น HO, WH1)
4.  **Hourly Rate**: ฐานในการคำนวณเงินเดือน
5.  **Accommodation & Followers**: จำเป็นมากสำหรับการหักค่าใช้จ่ายแคมป์คนงาน

### 📝 หมายเหตุการใช้งาน (Usage)
*   **Template CSV**: ใช้ `,` เป็นตัวคั่นสำหรับรหัสโครงการ (เช่น `HO,WH1`)
*   **การบันทึกไฟล์ (Save As)**: ใน Excel ให้เลือก Format เป็น **"CSV UTF-8 (Comma delimited) (*.csv)"** เพื่อป้องกันปัญหาภาษาไทยเป็นภาษาต่างดาว
*   **Database**: แยก Income/Expense เป็น Sub-collection เพื่อรองรับ History
